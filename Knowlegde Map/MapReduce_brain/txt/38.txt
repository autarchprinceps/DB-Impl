<h1>Search your Hadoop Data and Get Real-Time Results</h1>
<h2>Deep API integration makes searching data in Hadoop easy</h2>
</hgroup>

<td valign="top" width="47%">
<ul style="padding-bottom: 40px;">
<li>
<h3>Great fit for &#8220;Big Data&#8221; and Hadoop</h3>
<p>Elasticsearch’s distributed nature allows it to search ­ and store ­ vast amounts of information in near real­time.</li>
<li>
<h3>Rich Language to Ask Better Questions</h3>
<p>Elasticsearch provides a rich language for users to ask better questions in order to get clearer answers, significantly faster.</li>
<li>
<h3>Leave Search to the Search Engine and Focus on Data Transformation</h3>
<p>Elasticsearch is natively integrated with Hadoop so there is no gap for the user to bridge; we provide dedicated Input and OutputFormat for vanilla Map/Reduce, Taps for reading and writing data in Cascading, and Storages for Pig and Hive so you can access Elasticsearch just as if the data were in HDFS.</li>
<li>
<h3>Scale Your Hadoop Cluster Alongside Elasticsearch</h3>
<p>Distributed nature of the Map/Reduce model fits really well on top of Elasticsearch to correlate the number of Map/Reduce tasks with the number of Elasticsearch shards.</li>
<li>
<h3>Documentation</h3>
<p>Get <a href="http://www.elasticsearch.org/guide/en/elasticsearch/hadoop/current/index.html">documentation</a> for Elasticsearch for Apache Hadoop support. If that isn’t sufficient, you can email the mailing list. See the <a href="/community/">community page</a> for all your options.</li>
</ul>
</td>
<td valign="top" width="6%"></td>
<td valign="top" width="47%">
<ul style="padding-bottom: 40px;">
<li>
<h3>Enhanced Workflow</h3>
<p>Elasticsearch enables Hadoop users (including Map/Reduce, Hive, Pig and Cascading) to enhance their workflow with a full­blown search engine.</p>
<div class="workswith-logos"><img alt="" src="/content/uploads/2013/09/logos-hadoop-workswith.jpg" /></div>
</li>
<li>
<h3>Eliminate Network Traffic and Improve Performance</h3>
<p>Integration enables cluster co­locations by exposing shard information to Hadoop. Job tasks are run on the same machines as the Elasticsearch shards themselves, eliminating network traffic and improving performance through data locality.</li>
<li>
<h3>Real-time Responses Improve Job Execution and Cost</h3>
<p>Elasticsearch provides near real­time responses (think milliseconds) that significantly improve a Hadoop job’s execution and the cost associated with it, especially when running on ‘rented resources&#8217; such as Amazon EMR.</li>
</ul>
</td>
</tr>
